# 專案規則

## 每次寫完程式後必做

每次完成程式修改並 push 之後，**必須**更新 `LAST_CHANGES.md`（位於此目錄根層）。

格式如下，用繁體中文描述，不要只寫 commit 訊息：

```
## 最後一次修改（YYYY-MM-DD）

### 改了什麼
- （具體描述每個修改項目，例如：「修正撞牆死亡時頭部屍體座標超出地圖邊界的問題」）

### 為什麼改
- （說明原因，例如：「Phase 2 先延伸頭部座標，Phase 3 才偵測碰牆，導致頭已在界外才死亡」）

### 影響的檔案
- `路徑/檔名.js` — 一行說明
```

規則：
- **累加，不覆蓋**：把新的修改區塊加在檔案**最上面**，舊的內容保留在下方
- 用開發者能看懂的中文，不要寫廢話
- 改了幾個功能就寫幾條，不要合併成一條
